# Optimize-StoragePool

## SYNOPSIS
Optimizes a Storage pool.

## SYNTAX

### ByFriendlyName (Default)
```
Optimize-StoragePool [-FriendlyName] <String[]> [-CimSession <CimSession[]>] [-ThrottleLimit <Int32>] [-AsJob]
 [-PassThru] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByUniqueId
```
Optimize-StoragePool -UniqueId <String[]> [-CimSession <CimSession[]>] [-ThrottleLimit <Int32>] [-AsJob]
 [-PassThru] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByName
```
Optimize-StoragePool -Name <String[]> [-CimSession <CimSession[]>] [-ThrottleLimit <Int32>] [-AsJob]
 [-PassThru] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObject (cdxml)
```
Optimize-StoragePool -InputObject <CimInstance[]> [-CimSession <CimSession[]>] [-ThrottleLimit <Int32>]
 [-AsJob] [-PassThru] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRIPTION
The **Optimize-StoragePool** cmdlet rebalances the Spaces allocations in a pool to disks with available capacity.

If you are adding new disks or fault domains, this operation helps move existing Spaces allocations to them, and optimization improves their performance.
However, rebalancing is an I/O intensive operation.

## EXAMPLES

### Example 1: Optimize a Storage pool
```
PS C:\>Optimize-StoragePool -FriendlyName "SQLPool"
```

This command rebalances the Spaces allocation for all of the Spaces in the pool named SQLPool.
